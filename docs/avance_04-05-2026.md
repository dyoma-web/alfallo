### 14b — Rediseño multi-meta por mes (`d75b64d`)

Cambio de modelo: una sola meta no alcanza. Se crea hoja nueva `metas_profesional`:

```
[id, profesional_id, periodo, nombre, tipo, valor, created_at, updated_at]
```

- Varias metas por mes con nombre libre.
- Tipos: `economica`, `usuarios`, `otra`.
- Constraint lógica: `(profesional_id, periodo, nombre)` único.
- Tier se calcula sobre la **suma de todas las metas tipo `economica`** del periodo.
- Endpoints CRUD: `metasListMine`, `metasCreate`, `metasUpdate`, `metasDelete` en nuevo `apps-script/metas.gs`.

### Fix: periodo Date→string (`4e968b8`)

Sheets convertía automáticamente `"2026-05"` a un objeto `Date` al guardarlo, lo que rompía el lookup `m.periodo === "2026-05"`. Se agregó:

- `metas_periodoToString_(v)` — normaliza Date e ISO string a `"YYYY-MM"`.
- `metas_ensurePeriodoTextFormat_()` — fija el número-formato de la columna a `@` (texto) para evitar conversiones futuras.

Logging de debug eliminado en `2c2cdbd`.

### MetasCard — curva S y marcador de hoy (`a96be61`)

En el dashboard del trainer:

- Cálculo de `expected = (metaEconomica * dayOfMonth) / daysInMonth`.
- Delta entre acumulado real y esperado.
- Marcador vertical "hoy" sobre la barra de progreso (`<TodayMark percent={...} />`).

### Histórico — navegación de periodo (`fd72443`)

En `/perfil`, controles `← Hoy →` para revisar metas de meses pasados/futuros sin recargar.

```ts
function shiftPeriod(period: string, delta: number): string {
  const m = /^(\d{4})-(\d{2})$/.exec(period);
  if (!m) return period;
  const date = new Date(Number(m[1]), Number(m[2]) - 1 + delta, 1);
  return `${date.getFullYear()}-${String(date.getMonth() + 1).padStart(2, '0')}`;
}
```

## 4. Calendario — fixes visuales

### `@import` antes de `@tailwind` (`3faf853`)

Bug crítico: `@import './styles/fullcalendar.css'` estaba **después** de `@tailwind base/components/utilities` en `src/index.css`. Por spec CSS, los `@import` posteriores se ignoran silenciosamente, lo que dejaba el calendario sin tema custom (líneas blancas prominentes, look genérico de FullCalendar).

Movido al inicio del archivo. El tema custom ahora aplica.

### Sin scroll interno en dashboard (`df7fd7a`)

`height={500}` → `height="auto"` en el calendario embebido del TrainerDashboard. Evita el scrollbar interno y deja que FullCalendar se dimensione al contenido.
